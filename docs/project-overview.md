# 项目概览 — 局域网服务统一代理工作台

## 产品定位

一个轻量级的局域网 Web 服务统一代理工作台，通过单一端口管理和访问内网所有 Web 服务。适合个人或单管理员场景，无需多用户体系。

## 核心体验

用户只需一个公网地址，即可安全、无缝地访问所有内部 Web 服务，切换流畅，界面统一。默认走代理嵌入；对依赖浏览器本地状态且必须复用原地址配置的服务，可配置为直连打开。

## 技术选型理由

| 技术 | 选择理由 |
|------|----------|
| Vue 3 | 组合式 API、响应式系统、生态成熟 |
| Tailwind CSS | 快速开发、暗色模式支持、无运行时开销 |
| Express | 轻量、稳定、中间件生态丰富 |
| http-proxy-middleware | 成熟的代理方案，支持响应拦截和重写 |
| JSON 文件存储 | 单用户场景足够，无需数据库 |

## 访问模式

- 代理嵌入：通过 `/<slug><entryPath>` iframe 加载目标服务，地址栏保持总端口。
- 直连打开：跳转到 `协议://目标IP:目标端口<entryPath>`，用于复用目标 origin 下的 localStorage、service worker、登录态或本地配置。
- 浏览器按 origin 隔离本地状态；`http://10.30.0.22:9090` 与 `http://10.30.0.22:39197` 不能共享 localStorage。

## 端口约定

| 用途 | 默认端口 | 说明 |
|------|----------|------|
| 主服务 | 39197 | 对外暴露的总端口，避开常见开发端口和全局 `PORT` 干扰 |
| 前端开发 | 5173 | 开发模式热更新端口 |

## 安全边界

- 仅限局域网 / 单管理员使用，无认证体系
- 代理层过滤内网信息头（X-Forwarded-For、X-Powered-By）
- 快捷操作需二次确认
- 不存储任何密钥或凭证
- 直连打开会暴露目标原始地址；仅在必须复用目标 origin 本地状态时启用。

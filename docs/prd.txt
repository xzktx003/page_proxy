## 1. 项目概述

### 1.1 项目名称
局域网服务统一代理工作台

### 1.2 核心目标
通过唯一对外暴露的总端口，统一管理和代理访问局域网内多台机器上的 Web 服务，避免直接暴露内网地址与端口。实现一键切换、全屏无感知访问，地址栏始终保持总端口域名，无跳转泄露。

### 1.3 目标用户
个人或单管理员使用，无需多用户体系。

### 1.4 运行环境
- 总端口服务部署在局域网内，通过路由器端口转发/反向代理等方式将单一公网端口映射至该服务。
- 被代理服务可以部署在同一台机器或局域网内其他机器。

## 2. 功能需求

### 2.1 服务管理

#### 2.1.1 添加服务
**触发**：首页“添加服务”按钮。

**表单字段**：
| 字段 | 类型 | 必填 | 限制/说明 |
|------|------|------|-----------|
| 服务名称 | 文本 | 是 | 最多 20 字，用于卡片标题、导航栏显示 |
| 服务标识 | 文本 | 否 | 唯一 ID，字母数字与连字符，不填则由系统自动生成 UUID；一经创建不可修改 |
| 目标 IP | 文本 | 是 | 支持局域网 IP 或 `127.0.0.1` |
| 目标端口 | 数字 | 是 | 1-65535 |
| 服务协议 | 下拉 | 是 | `http` / `https`，默认 `http` |
| 健康检查路径 | 文本 | 否 | 用于连通性检测的相对路径，默认 `/`；若目标服务根路径不响应 GET 可自定义 |
| 卡片图标 | 上传/内置图标库 | 否 | 未设置时默认展示服务名称首字 |
| 服务描述 | 文本 | 否 | 最多 50 字，鼠标悬停卡片时显示 |
| 所属分组 | 下拉/新建 | 否 | 未选择则归入“未分组” |

**后端处理**：
- 若用户未填服务标识，生成 UUID 作为唯一标识。
- 持久化校验目标服务可达性：依据 `协议 + IP + 端口 + 健康检查路径` 发送 GET 请求，超时时间 3 秒。若不可达，前端提示“目标服务无法访问，是否仍要添加？”允许强制添加（可后期手动上线）。
- 服务配置保存至本地 JSON 文件 `./config/services.json`。
- 添加成功后，前端即时刷新服务卡片列表。

#### 2.1.2 编辑服务
- 每个服务卡片右上角提供“编辑”按钮，点击弹出预填表单。
- 可修改除“服务标识”外的所有字段。修改后更新 JSON 配置文件，代理路由实时生效（无需重启服务）。
- 若修改目标地址或端口，重新触发连通性检测，处理逻辑与添加时一致。

#### 2.1.3 删除服务
- 卡片右上角提供“删除”按钮，点击弹出二次确认对话框。
- 确认后从 JSON 配置中删除对应条目，首页卡片同步移除，相关代理路由立即失效。

#### 2.1.4 服务分组管理
- 首页顶部显示分组筛选栏，包含“全部服务”及所有已存在的分组标签。
- 点击分组名称，下方仅展示该分组内的服务卡片。
- 支持通过拖拽卡片到分组标签的方式改变分组，拖拽结束后更新 JSON 配置中的分组字段。
- 在编辑服务表单中亦可直接选择/新建分组。

---

### 2.2 首页卡片工作台

#### 2.2.1 卡片样式
- 响应式网格布局：默认每行 4 列，窗口缩小时自动调整列数。
- 卡片基础样式：白色背景，圆角 8px，阴影 `0 1px 4px rgba(0,0,0,0.1)`，hover 时阴影加深并轻微上浮。
- 卡片内容自上而下：图标（或首字）、服务名称、运行状态标识（绿色圆点 + “在线” / 灰色圆点 + “离线”）。
- 状态检测：每 30 秒对 `健康检查路径` 发起一次 HEAD/GET 请求，更新在线状态。

#### 2.2.2 卡片交互
- **点击卡片**：跳转至 `/服务标识` 进入全屏嵌入模式。
- **卡片快捷操作**（在状态旁显示）：
  - 提供可配置的操作按钮，默认无操作。管理员可在服务编辑表单中自定义操作：
    - 操作名称、HTTP 方法、目标路径（相对目标服务）、确认提示。
    - 示例：重启服务 → `POST /restart`，清空缓存 → `POST /clear-cache`。
  - 若服务未配置操作，则隐藏该区域。
- **分享**：卡片底部提供分享图标，点击复制 `https://<公网总域名>/服务标识` 到剪贴板并提示。

#### 2.2.3 卡片排序
- 支持拖拽卡片调整显示顺序，排序序号持久化至 `services.json` 中每个服务的 `sort` 字段。
- 分组筛选下仍保持各自排序，不同分组的排序独立。

---

### 2.3 全屏嵌入与代理路由

#### 2.3.1 全屏嵌入页面布局
访问 `/服务标识` 时：
- 顶部固定导航栏（高度 40px，背景色 `#f5f5f5`）：
  - 左侧：返回首页按钮 + 当前服务名称。
  - 中间：服务切换下拉菜单（列出所有已添加服务，选中后直接跳转）。
  - 右侧：分享按钮。
- 下方 `<iframe>` 占据剩余 100% 宽度和高度，无边框，加载目标服务根路径。

#### 2.3.2 动态代理路由规则
- 所有发往 `/服务标识/...` 的请求，通过代理转发至 `协议://目标IP:目标端口/...`。
- **链接重写**：
  - 目标服务响应内容中的绝对地址（指向内网地址）被重写为 `/服务标识/...`。
  - 相对路径自动补全正确前缀。
- **跨域处理**：代理层添加 CORS 头，允许总端口域下的请求。
- 地址栏始终显示 `/服务标识/子路径`，不暴露任何内网信息。

#### 2.3.3 外部分享访问
- 用户通过复制的链接（`https://公网总域名/服务标识`）可直接进入对应服务的全屏嵌入模式。
- 无需额外认证。

### 2.4 个性化设置

#### 2.4.1 主题切换
- 首页导航栏右侧提供“浅色/深色”切换按钮。
- 主题状态存储于浏览器 localStorage，下次访问自动应用。

## 3. 非功能需求

### 3.1 性能
- 首页卡片列表加载时间 ≤ 1 秒。
- 代理转发额外延迟 ≤ 300ms。
- 状态检测请求对目标服务影响最小化（HEAD 优先）。

### 3.2 可靠性
- 修改服务配置后，代理路由生效时间 ≤ 2 秒，无需重启总端口服务。
- 服务配置文件损坏或格式错误时，后端应提供降级方案（如备份恢复），并输出错误日志。

### 3.3 兼容性
- 浏览器支持：Chrome 90+、Edge 90+。

### 3.4 安全性
- 代理层过滤掉目标服务响应中可能泄露内网地址的头部（如 `X-Forwarded-For` 可保留但应去除内网 IP）。
- 可配置的快捷操作需要二次确认，防止误操作。

| 层级 | 技术栈 | 说明 |

| 后端 | Node.js + Express + http-proxy-middleware | 动态路由代理、HTML 链接重写 |

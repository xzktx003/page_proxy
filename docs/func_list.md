# 功能清单 — 局域网服务统一代理工作台

| # | 功能模块 | 功能点 | 状态 | 说明 |
|---|----------|--------|------|------|
| 1 | 服务管理 | 添加服务 | ✅ | 表单包含：名称、标识、IP、端口、协议、健康检查路径、颜色、描述、分组、快捷操作 |
| 2 | 服务管理 | 编辑服务 | ✅ | 预填表单，除标识外均可修改，修改后路由实时生效 |
| 3 | 服务管理 | 删除服务 | ✅ | 二次确认对话框，确认后删除并移除路由 |
| 4 | 服务管理 | 服务标识自动生成 | ✅ | 未填标识时自动生成 UUID 前8位 |
| 5 | 服务管理 | 连通性检测 | ✅ | 添加/编辑时向目标 GET 请求，3秒超时，不可达可强制添加 |
| 6 | 服务管理 | JSON 持久化 | ✅ | 配置保存至 `config/services.json`，含备份恢复机制 |
| 6a | 服务管理 | 默认访问路径 | ✅ | 服务可配置点击卡片和分享链接使用的入口路径，例如 `/ui/#/proxies` |
| 6b | 服务管理 | 打开方式 | ✅ | 支持代理嵌入或直连打开；直连用于复用目标地址下的浏览器本地配置 |
| 7 | 服务分组 | 分组筛选 | ✅ | 首页顶部显示分组标签，点击筛选对应服务卡片 |
| 8 | 服务分组 | 新建/选择分组 | ✅ | 添加/编辑表单中支持输入新分组或从已有分组选择 |
| 9 | 首页卡片 | 响应式网格布局 | ✅ | 默认4列，响应式缩放（1/2/3/4列） |
| 10 | 首页卡片 | 卡片样式 | ✅ | 白色背景、圆角、阴影、hover 效果 |
| 11 | 首页卡片 | 状态检测 | ✅ | 每30秒 HEAD/GET 请求健康检查路径，显示在线/离线 |
| 11a | 首页卡片 | HTTPS 健康检查 | ✅ | 根据 service.protocol 选择 http/https 模块发起健康检查，HTTPS 目标容忍自签名证书 |
| 12 | 首页卡片 | 卡片点击跳转 | ✅ | 代理模式点击进入 `/服务标识` 全屏嵌入；直连模式点击进入目标原始地址 |
| 13 | 首页卡片 | 快捷操作按钮 | ✅ | 可配置操作名称、HTTP方法、路径、确认提示 |
| 14 | 首页卡片 | 分享链接 | ✅ | 卡片底部分享图标，复制链接到剪贴板 |
| 15 | 首页卡片 | 卡片排序 | ⚠️ | 数据模型已支持 sort 字段，UI 拖拽排序待完善 |
| 16 | 全屏嵌入 | 导航栏 | ✅ | 40px 高度，返回首页、服务名称、状态指示、服务切换下拉、分享、全屏按钮 |
| 17 | 全屏嵌入 | iframe 代理 | ✅ | iframe 占满剩余空间，加载代理内容 |
| 18 | 代理路由 | 动态代理转发 | ✅ | `/标识/...` → `协议://IP:端口/...` |
| 18a | 代理路由 | HTTPS 目标支持 | ✅ | 支持 HTTPS 协议目标服务，容忍自签名证书（secure: false / rejectUnauthorized: false） |
| 19 | 代理路由 | HTML 链接重写 | ✅ | 绝对地址、协议相对、根相对链接自动重写为代理路径 |
| 20 | 代理路由 | CSS url() 重写 | ✅ | CSS 中的 url() 引用自动补全代理前缀 |
| 21 | 代理路由 | CORS 头 | ✅ | 代理层添加 CORS 允许头 |
| 22 | 代理路由 | 隐藏内网信息 | ✅ | 过滤 X-Forwarded-For、X-Powered-By 等头 |
| 23 | 代理路由 | 错误页面 | ✅ | 目标不可达时返回友好错误页面 (502)，避免 iframe 空白 |
| 24 | 外部分享 | 直接访问 | ✅ | 分享链接可直接进入全屏嵌入模式 |
| 25 | 个性化 | 主题切换 | ✅ | 浅色/深色切换，localStorage 持久化 |
| 26 | 非功能 | 局域网可见 | ✅ | 服务绑定 0.0.0.0，默认端口 39197，前端 dev 模式 --host |
| 27 | 非功能 | 配置热更新 | ✅ | 修改配置后代理路由立即生效，无需重启 |
| 28 | 非功能 | 配置损坏恢复 | ✅ | JSON 解析失败时从 .bak 备份恢复 |
| 29 | 交付物 | 一键启动脚本 | ✅ | `start.sh` 自动安装依赖、构建、启动 |
| 30 | 交付物 | README | ✅ | 包含部署说明、端口转发、启动步骤 |
